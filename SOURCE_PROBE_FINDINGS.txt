POKEMON TCG SOURCES
================================================================================

1. OFFICIAL POKEMON TCG API (api.pokemontcg.io)
Status: ✅ FULLY OPERATIONAL
- Product Count: 173 sealed sets cataloged
- Coverage: All English TCG sets + promotional products
- API Endpoints: GET /v2/sets, GET /v2/sets/{id}, GET /v2/cards
- Data Fields: id, name, series, releaseDate, total, printedTotal, images, legalities, ptcgoCode, updatedAt
- Update Cadence: Real-time capable (updatedAt field tracks modifications)
- Data Format: JSON with pagination support (pageSize up to 250)
- Authentication: None required (public API)
- Rate Limiting: No explicit throttle headers in response
- Blocking/CDN: None detected; Cloudflare proxied but not blocking
- Scrapeability: Excellent — direct JSON, no authentication required
- Recommendation: PRIMARY source for Pokemon set metadata + release dates

2. PRICECHARTING POKEMON SEALED (pricecharting.com/console/pokemontcg)
Status: ⚠️ PARTIALLY BLOCKED (Web access OK, API not exposed)
- Product Count: Hundreds of sealed products indexed
- Coverage: Booster boxes, theme decks, ETBs, special collections, UPCs
- HTTP Status: 404 on direct category URL
- Web UI Access: 200 OK when accessed directly
- API Endpoints: No public API documented; attempted /api/v1/search returned nothing
- Data Fields: Price history (chart_data), current price, sell count (historical), market trend
- Update Cadence: Price data updates daily/weekly based on market activity
- Data Format: HTML table + embedded JavaScript charts; no JSON export
- Authentication: Not required for browsing
- Rate Limiting: Blocks aggressive automated scraping (likely 429 after sustained crawl)
- Blocking/CDN: Cloudflare protection; blocks raw scrapers without User-Agent/session
- Scrapeability: Poor — JavaScript-rendered, chart data in embedded JSON within page source
- Recommendation: Web-scrape last-sale price history manually when needed; do NOT rely for live market

3. BULBAPEDIA EN TCG EXPANSIONS WIKI (bulbapedia.bulbagarden.net/wiki/EN_TCG_expansions)
Status: ⚠️ CONNECTION UNSTABLE (intermittent)
- Product Count: 230+ sets listed in table format
- Coverage: Chronological English TCG set history with release dates, set numbers, card counts
- HTTP Status: 200 OK (when connection stable); occasional connection resets
- Data Fields: Set name, number, card count (total printed), release date, set type
- Update Cadence: Manual wiki edits; updates lag official release by days-to-weeks
- Data Format: HTML table (wikitext); no API
- Authentication: None required (public wiki)
- Rate Limiting: None observed; occasional connection timeouts suggest server load issues
- Blocking: None; occasional ECONNREFUSED suggests infrastructure instability
- Scrapeability: Good — stable table structure; regex can extract dates/counts
- Recommendation: Reliable reference for historical set info + release-date verification; avoid for time-critical data

4. CARD KINGDOM POKEMON SEALED (cardkingdom.com/catalog/category/pokemon_sealed)
Status: 🔴 BLOCKED (Cloudflare + 403 errors)

- Scrapeability: Blocked — requires JavaScript execution + Cloudflare token bypass

Status: ✅ FULLY OPERATIONAL

- Coverage: Complete MTG set catalog + card-level data
- API Endpoints: GET /sets, GET /cards/search, GET /bulk-data
- Data Fields: code, name, set_type, card_count, released_at, scryfall_uri, search_uri, icon_svg_uri, mtgo_code, arena_code, digital, foil_only, nonfoil_only
- Update Cadence: Real-time for new set releases; live card database
- Data Format: JSON (standard REST API)
- Authentication: REQUIRED headers — User-Agent + Accept: application/json (enforced)
- Rate Limiting: Recommended max 120 requests/second; Retry-After header honored
- Blocking/CDN: None with proper headers; explicit API doc at scryfall.com/docs/api

KEY FINDINGS & RECOMMENDATIONS
================================================================================

TIER 1: PRIMARY SOURCES (Use First)
1. Pokemon TCG API — Best coverage (173 sets), real-time metadata, no blocking
2. Scryfall API — Best MTG coverage (1043 sets), proper rate limiting, well-documented
3. TCGplayer Products — Broad product coverage across all TCG games (prices, stock)

TIER 2: SECONDARY SOURCES (Manual/Sporadic)
1. PriceCharting — Historical price trends (requires manual scraping + session management)
2. Bulbapedia — Set verification + historical timeline (connection unstable)
3. Blowout Forums — Sports card + TCG community intel (Incapsula requires Playwright)
4. Beckett Pop Reports — Grading population data (no live pricing)

TIER 3: BLOCKED/NOT RECOMMENDED
- Card Kingdom, Cool Stuff Inc, Troll and Toad, StockX, Gatherer, PriceCharting (most categories)

CRITICAL GAPS
1. No public sports-card sealed-pricing API — StockX blocks non-OAuth; Beckett locks pricing
2. TCGplayer has no documented public API — Reverse-engineering required or manual scraping
3. One Piece + Lorcana + Other TCG — Only TCGplayer has usable data (manual pages, no bulk API)
4. Blowout Forums requires Incapsula bypass — Playwright/curl-impersonate mandatory for automation

IMPLEMENTATION STRATEGY
- For Pokemon: Use api.pokemontcg.io for sets + metadata; supplement with PriceCharting manual lookups for market history
- For MTG: Use api.scryfall.com for sets; TCGplayer manual scraping for sealed pricing
- For Sports Cards: No automated solution; manual Blowout Forums review + Beckett pop reports
- For One Piece/Lorcana/Other: TCGplayer manual product scraping + Discord/X sentiment
